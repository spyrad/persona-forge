### Befunde / Gotchas

- **`unfinished`/`empty` deterministisch erzeugen:** `pending`-Lauf via `POST /api/runs` ohne Step-Loop (bleibt `pending`); `empty` via Bad-Key-Config (alle reps `error` → 0 verwertbar). Ein N=5-Lauf wird gegen die echte OpenAI-API zu schnell `completed`, um `running` einzufangen.
- **Logout-Redirect ohne Session-Verlust testen:** `fetch(url, {credentials:'omit', redirect:'follow'})` aus der eingeloggten Konsole → Middleware redirectet zu `/auth/signin`, ohne den eingeloggten Tab auszuloggen.
- **Playwright-MCP-Browser ≠ Damians Browser:** der MCP-gesteuerte Browser hat eine eigene Session; für den RLS-Test muss der zweite Account **in diesem** Browser eingeloggt werden (A ausloggen → B einloggen).
- **`git mv` am Windows-Lock** (Permission denied) → `Move-Item` + `git add <alt> <neu>` (Rename bleibt erhalten). **Bash-Tool ≠ PowerShell:** Here-String `@'…'@` ist in Git Bash kein Here-String → Commit-Message per `git commit -F -` mit POSIX-Heredoc.
