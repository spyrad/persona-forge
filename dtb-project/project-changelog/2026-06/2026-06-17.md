- **Phase 1 (Freitext-Durchstich)** — committet `0f62896`, Manual-Gate abgenommen:
  - Migration `personas` auf **Prod appliziert** (`db push`, autorisiert): RLS `own-or-global`,

  - Service `personas.ts` (list/create/duplicate/delete, **kein** update), Types (Persona/PersonaView/
    CreatePersonaInput), CRUD-API `/api/personas` (GET/POST, DELETE, **POST [id]/duplicate**, **kein PUT**),
    React-Island `PersonaCatalog` (Freitext-Form, Katalog, Kopieren, Löschen, Tag-Filter),
    geschützte `/personas`-Page, `PROTECTED_ROUTES += /personas`, Dashboard-Link.
  - **Lint-Befund:** React-Compiler `immutability` flaggt `window.location.href =` in voll
    kompilierbaren Islands → in Modul-Scope-Helper `redirectToSignin()` ausgelagert (ModelConfigManager
    bailt aus anderen Gründen, daher dort nicht geflaggt).
- **Phase 2 (Strukturierter Spec-Editor)** — codiert, automated-grün, **uncommitted**:
  - Reine, getestete Kompilier-Funktion `persona-compile.ts` (Spec §§1–6 → Markdown-System-Prompt,
    deterministisch) + 7 Vitest-Tests (Tests gesamt 13→20).
  - Types: `PersonaStructuredFields`, diskriminierte `CreatePersonaInput` (freeform/structured),
    `PersonaView.structuredFields`. Service: `createPersona` verzweigt auf `sourceKind` (structured →
    serverseitig kompiliert + `structured_fields` gespeichert). API: `createSchema` als
    `z.discriminatedUnion("sourceKind", …)`.
  - Island: Modus-Umschaltung Freitext/Strukturiert (§§1–4 Pflicht-Listenfelder + §§5–6 optional).
  - **Entscheidung (Hybrid):** Plan-Spannung „Kopieren öffnet Formular" vs. Phase-1-Sofort-Duplicate
    aufgelöst → **„Kopieren"** bleibt Sofort-Duplicate, neuer **„Anpassen"**-Button befüllt das Formular
