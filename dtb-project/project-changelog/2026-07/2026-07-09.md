## Session 1 — CI-Review-Agent gebaut, verifiziert und deployt (Modul 5 / Champion)

### Implementiert

Der Change `ci-review-agent` ist **komplett** (4 Phasen, `status: implemented`) und
auf `main` deployt. Ein LLM-basierter PR-Reviewer läuft als eigener CI-Workflow,
prüft sechs projektspezifische Kriterien und gatet den Merge über einen Required
Status Check.

**Phase 1 (`fe00683`) — reiner Kern.** `src/lib/ai-review/`: `prepareDiff` (filtern →
priorisieren → kappen), zod-Schema, `decideVerdict` als deterministische Schwelle.
Kein LLM, kein Netz, voll unit-getestet.

**Phase 2 (`999c966`) — Scorer-Entry.** `scripts/ai-review.ts` mit Vercel AI SDK 7
(`ToolLoopAgent` + `Output.object`) gegen z.ai/GLM. `git diff | npx tsx` → JSON.

**Phase 3 (`f23e74c`, `95efbc0`, `7cadefe`) — CI-Verdrahtung.** Composite Action +
eigener Workflow `ai-review.yml`, getrennt von `ci.yml`. PR-Kommentar mit Dedup,
Labels `ai-cr:passed`/`ai-cr:failed`, Commit-Status `ai-review/verdict`.

**Phase 4 (`0a9346e`) — promptfoo + statische Regel-Prüfung.** `npm run eval:review`
fährt den echten Scorer gegen vier Fixtures (12/12 Assertions über 3 Wiederholungen).

### Entscheidungen

- **Findings statt Noten (Phase 2).** Gemessen (glm-5.2, `temperature: 0`, gleicher
  Diff dreimal): `apiQuartet` schwankte 3/8/8, das Verdict kippte `failed`→`passed`.
  Eine exakte Schwelle auf einer gewürfelten Zahl ist Scheinsicherheit. Das Modell
  meldet jetzt nur Regel-Verstöße mit Datei und Beleg; Schweregrad und Score leitet
  `verdict.ts` im Code ab.
- **Sieben Regeln deterministisch (Phase 4).** glm-5.2 übersah `missing-rls` in 1 von
  3 Läufen — bei einer Migration mit vorbildlichen Policies, der nur das
  `enable row level security` fehlte. Eine geschärfte Prompt-Beschreibung machte es
  **schlimmer** (0 von 5). Falsch-Negativ bei einem Sicherheits-Check. Konsequenz:
  `static-checks.ts` prüft die syntaktisch entscheidbaren Regeln per Regex
  (Trefferquote 5/5); das Modell sieht sie nicht mehr.
- **Eigener Workflow statt Job in `ci.yml`.** Ein z.ai-Ausfall darf den Prod-Deploy
  nie blockieren. Die Merge-Sperre entsteht über einen separaten Commit-Status.
- **Branch-Protection auf `main`** mit `ai-review/verdict` als Required Check,
  `enforce_admins: false` — der direkte Push-Deploy bleibt möglich.
- **promptfoo NICHT als devDependency** (`3306308`): es blähte den Lockfile um ~17.000
  Zeilen und wurde in jedem CI-Job mitinstalliert. Jetzt `npx -y promptfoo@0.121.18`.
- Kein Modellvergleich (nur z.ai/glm-5.2), kein Timeout im Scorer — beides bewusst,
  in `change.md` vermerkt.

### Bugs, die nur die Realität fand

- **`diff.ts` warf sich selbst aus dem Review**: die Binär-Erkennung prüfte den ganzen
  Block auf `GIT binary patch`, und der Quelltext enthält diesen String. Der Reviewer
  meldete grün über eine Datei, die er nie gelesen hatte. Marker zählen jetzt nur in
  Metazeilen (ohne `+`/`-`-Präfix).
- **SDK-Warnung landete auf stdout**, mitten im JSON, das die Action parst → via
  `AI_SDK_LOG_WARNINGS` nach stderr.
- **`process.exit()`** löste auf Windows einen libuv-Assert aus (Exit-Code 127 statt 1)
  → `process.exitCode`.
- **Diff-Priorisierung**: `scripts/*.ts` konkurrierte mit `plan.md` und
  `package-lock.json` um denselben Rang. Prosa und Manifeste rangieren jetzt hinter
  jedem Code; Budget 60k → 120k Zeichen.
- **`npm ci` gebrochen** durch den Phase-4-Push (Lockfile out of sync) → `ci` rot,
  `deploy` still übersprungen (genau das Muster aus `lessons.md`).
- **`eslint .` brach über `promptfoo/reviewProvider.js`** (`d4344f9`). Mein Fehler:
  lokal nur Teilmengen gelintet, weil das volle `npm run lint` im CRLF-Rauschen
  untergeht. Der CI-äquivalente lokale Check ist
  `npx eslint . --rule '{"prettier/prettier":"off"}'`.
- **`lint-staged` ließ `prettier --write` auf `package-lock.json` los** — stille Quelle
  von `npm ci`-Fehlern. Glob nimmt es jetzt aus.

## Session 2 — 19:21 — Landing-Page-Redesign „Live Instrument" (gebaut → gemergt → live)

### Implementiert

Die öffentliche Startseite `/` ist eine englischsprachige Showcase-Seite im Stil eines
Präzisions-Messinstruments. **PR #3 gemergt, auf Prod deployt** (`c04001b`), Smoke bestanden.

- **Hero mit Canvas-Live-Simulation**: ein Messlauf spielt sich ab — Punkte tröpfeln ein,
  stapeln sich zum Histogramm, eine Verteilungskurve entsteht, die Statuszeile tickt echt
  berechnete Werte (`run 37/50 · mean 62.4 · σ 8.1`). Nach N=50 wechselt die Achse
  (E/I → S/N → T/F → J/P). Genau eine React-Insel, reines Canvas-2D, keine Chart-Lib.
- **Fünf Sektionen**: Hero, Problem (Anekdote vs. Verteilung), Methode (Configure/Run/Read
  mit Mini-Diagrammen), Test Library, CTA + Footer.
- **Test Library ist zukunftsoffen**: OEJTS + Standhaftigkeit `live`, Big Five (Mini-IPIP) +
  Task-based evals `planned`. Ein neuer Test ist künftig eine Karte, kein Redesign.
- **Footer trägt die korrekte OEJTS-Attribution** (Eric Jorgenson, openpsychometrics.org,
  CC BY-NC-SA 4.0). Die falsche „gemeinfrei"-Behauptung ist damit von der Seite verschwunden.
- **Typografie**: Instrument Serif (Headlines) + JetBrains Mono (alle Zahlen, `tabular-nums`),
  self-hosted via `@fontsource`, nur auf der Landing geladen.
- Farben ausschließlich über semantische Tokens; der Canvas liest sie zur Laufzeit via
  `getComputedStyle` — Dark/Light folgen automatisch. Teal nur für lebende Messung,
  Amber genau einmal (der irreführende Einzelpunkt in der Problem-Sektion).

### Entscheidungen

- **Ansatz „Live Instrument" statt WebGL-Showcase.** Die Simulation _zeigt_ das
  Produktversprechen, statt es zu behaupten. Präzision passt besser zur Thematik als ein
  Partikelfeuerwerk — und bleibt ohne Three.js/Framer Motion schlank.
- **Seite englisch, App bleibt deutsch.** Bewusster Sprachbruch beim Login; Reichweite fürs
  Portfolio schlägt Konsistenz.
- **Teal auf den CTA-Buttons behalten** (User-Entscheidung): Primärbuttons sind App-Konvention;
  die einzigen Conversion-Elemente zu entfärben, um der Metapher zu genügen, wäre der falsche
  Trade. Ebenso `hover:border-primary/40` auf den Live-Karten („aktive Zustände").
- **`binValues` entfernt** statt totes Export mitzuschleppen (Final-Review-Fund).

### Bugs, die nur das Review fand

Vier echte Fehler, drei davon **schon im Plan-Code** — der Plan war nicht schlauer als der Reviewer:

- **Animation lief off-screen weiter**: `visibilitychange` rief `start()` ohne den
  Viewport-Status zu prüfen. Nach „wegscrollen → Tab wechseln → zurück" lief die rAF-Schleife
  unsichtbar durch. Fix: gemeinsames `isIntersecting`-Flag.
- **Mobiler Hero lief über**: nachgemessen bei 390 px — Inhalt 249,5 px gegen 240 px Box.
  Beide Kinder `absolute inset-0`, die Box wächst also nicht mit. Fix: `min-h-[17rem]`.
- **Ohne JS überlappte die SSR-Insel das Fallback** — zwei Statuszeilen übereinander.
  Fix: Insel startet `opacity-0`, derselbe Mount-Effekt blendet sie ein, der das Fallback versteckt.
- **Ohne JS waren die Sektionen 02–05 unsichtbar** (`.reveal { opacity: 0 }` unbedingt).
  Fix: Gating über eine `js`-Klasse; die reduced-motion-Regel musste in der Spezifität mitziehen.
